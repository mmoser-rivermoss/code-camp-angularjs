module.value

No dependency injection. The object that you give it is the object that you get when injecting into others.

module.service

Dependency injection allowed. It will new up the function that you give it.

module.factory

Dependency injection allowed. You will get back the service that the factory function returns. This allows you to determine which service to return if you'd like. Hence the word factory, but 99% of the time it isn't used this way.

module.provider

Dependency injection allowed. The service will be what the $get function returns. It allows for configuration at config time to set up whatever is needed for the service.
